# Claude Usage Widget

Claude Code の使用量をデスクトップに**常時表示**するフローティングウィジェット（Windows / Electron）。

<img src="resources/icon.png" width="72" alt="icon" />

開発中いつでも、画面の片隅で「あと5時間枠・週間枠をどれだけ使えるか／今日どれだけ使ったか」を一目で確認できます。Claude Code のターミナル風（暖色ダーク＋コーラル）デザイン。

---

## 表示される内容

ローカルの `~/.claude/projects/**/*.jsonl`（Claude Code のセッションログ）を集計し、半透明の小窓に表示します。

| パネル | 内容 |
| --- | --- |
| **利用枠（5時間 / 1週間）** | Claude のレート制限に対応する2つの枠の消費ゲージ（コーラル→黄→赤）。各枠の基準値は設定で変更可。 |
| **今日のトークン** | 今日のモデル別トークン（上位2モデル）＋メッセージ数 / セッション数。 |
| **推定コスト** | 公開単価で換算した今日 / 7日の推定USD。サブスク利用では課金されないため**参考値**。 |

> **重要**: 利用枠の数値は**ローカルログのトークン量から算出した推定**です。Claude 公式 `/usage` のプラン残量とは別物で、完全一致はしません（公式の残量データはローカルに保存されていないため）。基準値を自分の体感上限に合わせて調整して使ってください。

- **自動更新**: Claude Code が応答するたびに数秒で反映（ファイル監視＋30秒ごとの再集計）。
- **オフライン完結**: ログを読むだけ。ネットワーク通信・認証は一切不要。

## インストール（配布版を使う）

ビルド済みの配布ファイル（`release/` 内）をもらった場合は、どちらかをダブルクリックするだけ。**Node.js は不要**です。

| ファイル | 使い方 |
| --- | --- |
| `Claude Usage Widget Setup x.y.z.exe`（インストーラ） | ダブルクリックでインストール。スタートメニュー＆デスクトップにショートカットが作られ、インストール後すぐ起動。管理者権限不要（ユーザー単位）。 |
| `ClaudeUsageWidget-portable-x.y.z.exe`（ポータブル） | インストール不要。ダブルクリックでそのまま起動。USB や共有フォルダ向き。 |

**自動起動はデフォルトでON**です（OSログイン時に自動でウィジェットが立ち上がります）。不要ならトレイメニューまたは設定でOFFにできます。

## 自分でビルドする / 開発する

前提: [Node.js](https://nodejs.org/) 20 以上（Windows 11）。

```bash
cd C:\dev\projects\Claude-Usage-Widget
npm install
npm run dev        # ウィジェットが起動（開発モード・ホットリロード）
```

配布ファイルを作る:

```bash
npm run package    # release/ に インストーラ(.exe) と ポータブル(.exe) を生成
```

> **GUI が必要です。** ヘッドレス環境や `ELECTRON_RUN_AS_NODE=1` が設定された環境では起動できません（通常のデスクトップなら問題ありません）。

## 使い方

### ウィンドウ操作
- **移動**: ヘッダ（"✳ claude usage" の行）をドラッグ。位置は自動保存され、次回も同じ場所に出ます。
- **⟳ ボタン**: 即時更新。
- **⚙ ボタン**: 設定パネルの開閉。

### 設定（⚙ または トレイメニュー）
- **5時間 基準 (Mtok)** / **1週間 基準 (Mtok)**: 各ゲージが 100% になるトークン数（百万単位）。自分の体感上限に合わせて調整。
- **コスト表示**: 推定コストパネルの表示/非表示。
- **OS起動時に自動起動**: ログイン時の自動起動（デフォルトON）。

### システムトレイ
タスクトレイのアイコンを**右クリック**でメニュー、**左クリック**で表示/非表示。

| メニュー | 動作 |
| --- | --- |
| 表示 / 非表示 | ウィンドウの表示切替 |
| 常に最前面 | 他ウィンドウより手前に固定 |
| コスト表示 | コストパネルの表示切替 |
| OS起動時に自動起動 | ログイン時の自動起動切替 |
| 今すぐ更新 | 手動で再集計 |
| 終了 | アプリを終了 |

## 困ったとき

| 症状 | 対処 |
| --- | --- |
| 「Claude Code のデータが見つかりません」 | `~/.claude/projects/` が存在するか確認。別の場所なら環境変数 `CLAUDE_CONFIG_DIR` を設定。 |
| 数値が増えない | Claude Code を1ターン動かすと数秒で反映。されなければ ⟳ で手動更新。 |
| 利用枠の % が体感と合わない | 設定で「5時間 / 1週間 基準」を自分のプランの体感上限に合わせて調整。**公式 `/usage` と完全一致はしません**（ローカル推定のため）。 |
| コストが実際の請求と違う | 公開単価ベースの**参考値**。サブスク利用では実課金は発生しません。 |
| ウィンドウが見当たらない | トレイアイコンを左クリックで再表示。「常に最前面」をオンに。 |

## 開発者向け

```bash
npm run collect    # 集計ロジックを CLI で実行（Electron 不要、動作確認用。5時間/週間の両枠を出力）
npm run typecheck  # 型チェック
npm run build      # 本番ビルド（out/）
```

設計・アーキテクチャ・拡張方法・既知の制約は **[DESIGN.md](DESIGN.md)** を参照（実装の引き継ぎ用）。

単価は `src/main/usage/pricing.ts` に公開価格（2026-06時点）を定数化。価格改定時はこのファイルを更新してください。
